# تحويل النظام إلى تطبيق أندرويد

تم تجهيز طريقتين متكاملتين. الأولى تعمل فوراً بدون أي برامج، والثانية تنتج ملف APK حقيقي تحتاج بناءه على جهازك.

---

## ✅ المشروع مكتمل وجاهز للتجربة

كل الملفات المطلوبة (`www/firebase-config.js` و `www/modules/reports.js` وباقي الموديولات) موجودة الآن داخل هذه الحزمة. لا حاجة لأي إضافة يدوية — يمكنك المتابعة مباشرة للطريقة الأولى أو الثانية أدناه.

---

## 🔧 مشكلة "الأزرار لا تعمل والبيانات لا تظهر" داخل تطبيق APK

إن كنت جربت التطبيق ووجدت أن الأزرار لا تستجيب والبيانات لا تصل من Firebase (بينما نسخة PWA تعمل بشكل طبيعي في المتصفح)، فهذا يعني أن هناك خطأ برمجي يمنع تشغيل الصفحة بالكامل داخل WebView الخاص بأندرويد. السبب الأشهر لهذا الفرق تحديداً: بعض إصدارات WebView على أندرويد لا تُحمّل وحدات جافاسكريبت (ES Modules) من نطاق خارجي (مثل مكتبة Firebase من `gstatic.com`) بنفس موثوقية متصفح Chrome الكامل.

تم تطبيق إصلاحين في هذه الحزمة:

1. **طبقة عرض الأخطاء على الشاشة**: أضفت كوداً يعرض أي خطأ برمجي مباشرة كرسالة حمراء على شاشة الهاتف نفسه (بما أن تطبيق APK لا توجد فيه أدوات مطوّر ظاهرة). إن ظهرت هذه الرسالة، التقط لها لقطة شاشة وأرسلها لي لتحديد السبب بدقة متناهية.

2. **تحميل مكتبة Firebase محلياً عند البناء (للأندرويد فقط)**: عدّلت ملف `.github/workflows/build-apk.yml` بحيث يقوم بتنزيل ملفات مكتبة Firebase وتضمينها داخل التطبيق مباشرة بدلاً من تحميلها من الإنترنت في كل مرة يفتح فيها التطبيق — وهذا يُصلح الأمر إن كان السبب فعلاً هو تحميل الوحدات الخارجية. **هذا التعديل يخص نسخة APK فقط ولا يؤثر إطلاقاً على نسخة PWA** التي تعمل حالياً بنجاح.

### الخطوات التالية

1. ارفع الملفات المُحدّثة في هذه الحزمة إلى نفس مستودع GitHub (نفس طريقة الرفع السابقة، استبدل الملفات القديمة).
2. انتظر اكتمال البناء التلقائي من تبويب Actions، ونزّل ملف APK الجديد.
3. احذف نسخة التطبيق القديمة من الهاتف وثبّت النسخة الجديدة (لضمان عدم تعارض نسخ قديمة مخزّنة).
4. افتح التطبيق:
   - إن عملت الأزرار وظهرت البيانات ✅ تم حل المشكلة.
   - إن ظهرت رسالة حمراء بخطأ ⚠️ التقط لها لقطة شاشة وأرسلها لي فوراً لأحدد الإصلاح النهائي بدقة.
   - إن لم يظهر شيء إطلاقاً (شاشة بيضاء تماماً بدون أي رسالة) فالمشكلة غالباً صلاحية الإنترنت (`INTERNET permission`) — تحقق أنك لم تُعدّل ملف `android/app/src/main/AndroidManifest.xml` يدوياً (Capacitor يضيف هذه الصلاحية تلقائياً بشكل افتراضي).

---



لا تحتاج أي بناء أو متجر تطبيقات. الخطوات:

1. ارفع محتويات مجلد `www/` كما هي على استضافة تدعم **HTTPS** (Firebase Hosting، Netlify، Vercel، أو أي استضافة عادية بشهادة SSL). PWA لا تعمل بدون HTTPS.
2. افتح الرابط من متصفح Chrome على هاتف أندرويد.
3. سيظهر تلقائياً زر **"📱 تثبيت التطبيق على الهاتف"** أسفل يسار الشاشة (أو من قائمة Chrome ⋮ ← "تثبيت التطبيق" / "إضافة إلى الشاشة الرئيسية").
4. بعد التثبيت، يفتح التطبيق بأيقونته الخاصة بملء الشاشة تماماً مثل أي تطبيق أندرويد آخر، بدون شريط عنوان المتصفح.

**الطباعة والتصدير في هذه الطريقة:**
- زر "طباعة" يفتح مباشرة نافذة الطباعة الأصلية لأندرويد (حفظ PDF / اختيار طابعة).
- أزرار "تصدير PDF / PNG" تفتح **قائمة المشاركة الأصلية لأندرويد** (نفس القائمة التي تراها عند مشاركة صورة من أي تطبيق) والتي تتضمن خيار "طباعة" مباشرة، بالإضافة إلى المشاركة عبر واتساب/البريد/حفظ في الملفات. كل هذا يحدث **داخل التطبيق** دون فتح متصفح خارجي.

## الطريقة الثانية: تطبيق أندرويد حقيقي APK (عبر Capacitor)

هذا ينتج تطبيقاً مثبّتاً بالكامل (يمكن رفعه لاحقاً على Google Play أيضاً). هناك طريقتان لبنائه:

- **(أ) بناء سحابي عبر GitHub Actions** — الأسهل، لا تحتاج تثبيت أي برنامج على جهازك إطلاقاً.
- **(ب) بناء محلي عبر Android Studio** — يحتاج تثبيت برامج على جهازك، لكنه مفيد إن أردت تطوير/تعديل التطبيق لاحقاً.

---

### (أ) البناء السحابي عبر GitHub Actions — بدون تثبيت أي شيء

تم تجهيز ملف `.github/workflows/build-apk.yml` داخل هذه الحزمة يقوم ببناء ملف APK تلقائياً على خوادم GitHub المجانية. كل ما تحتاجه حساب GitHub مجاني ومتصفح:

1. أنشئ حساباً مجانياً على [github.com](https://github.com) إن لم يكن لديك واحد.
2. أنشئ مستودعاً (Repository) جديداً فارغاً (اجعله Private إن أردت خصوصية أكبر، خصوصاً لأن `firebase-config.js` بداخله).
3. **⚠️ مهم جداً (سبب شائع لفشل البناء):** فك ضغط الملف المرفق أولاً على جهازك. ثم من صفحة المستودع في GitHub اضغط **Add file → Upload files**، وادخل إلى المجلد الذي فككت ضغطه، **حدد كل ما بداخله** (`www`، `.github`، `package.json`، `capacitor.config.json`، `README.md`، `.gitignore`) واسحبها كلها معاً، ثم أفلتها، ثم Commit.
   لا تسحب المجلد الخارجي نفسه كوحدة واحدة — يجب أن يظهر `package.json` و `www` مباشرة في جذر المستودع (الصفحة الرئيسية للمستودع)، وليس داخل مجلد فرعي. إن كان الملف المضغوط اسمه `android-app.zip` ويحتوي مجلداً باسم `android-app` من الداخل، تجاهل ذلك الاسم وادخل بداخله وارفع محتوياته فقط.
   (لطمأنتك: أضفت أيضاً خطوة تحقق تلقائية في `build-apk.yml` تكتشف هذا الخطأ وتصلحه تلقائياً إن حدث لاحقاً بالخطأ.)
4. بمجرد الرفع، افتح تبويب **Actions** أعلى صفحة المستودع — سيبدأ البناء تلقائياً (يستغرق عادة 3-5 دقائق). يمكنك أيضاً تشغيله يدوياً من هناك عبر زر **Run workflow**.
5. بعد اكتمال البناء (علامة ✅ خضراء)، افتح تفاصيل التشغيل (Run) ثم انزل لقسم **Artifacts** أسفل الصفحة، واضغط لتنزيل `abu-mohammed-app-apk` — وهو ملف مضغوط يحتوي `app-debug.apk` جاهزاً للتثبيت المباشر على أي هاتف أندرويد (فعّل "مصادر غير معروفة" عند التثبيت اليدوي).

كرر الخطوتين 3-5 (رفع الملفات المُعدّلة ثم تنزيل الـ Artifact الجديد) في كل مرة تريد نسخة محدثة من التطبيق.

---

### (ب) البناء المحلي عبر Android Studio

يتطلب تجهيز جهاز بالبرامج التالية **مرة واحدة فقط**:

#### المتطلبات (تُثبّت مرة واحدة على جهاز الكمبيوتر)
- [Node.js](https://nodejs.org) (نسخة 18 فأعلى)
- [Android Studio](https://developer.android.com/studio) (يشمل Android SDK تلقائياً)
- Java JDK 17 (يأتي غالباً مضمّناً مع Android Studio)

#### خطوات البناء

1. افتح Terminal / موجه الأوامر داخل مجلد `android-app` (هذه الحزمة نفسها).

2. ثبّت الحزم:
   ```bash
   npm install
   ```

3. أضف منصة أندرويد (يُنشئ مجلد `android/` تلقائياً):
   ```bash
   npx cap add android
   ```

4. زامن ملفات الويب (`www/`) مع المشروع الأندرويدي — كرر هذه الخطوة في كل مرة تُعدّل فيها الملفات:
   ```bash
   npx cap sync android
   ```

5. افتح المشروع في Android Studio:
   ```bash
   npx cap open android
   ```

6. من داخل Android Studio: انتظر اكتمال المزامنة (Gradle Sync)، ثم من القائمة:
   `Build → Build Bundle(s) / APK(s) → Build APK(s)`

7. ستجد ملف `app-debug.apk` جاهزاً للتثبيت المباشر على أي هاتف أندرويد (فعّل "مصادر غير معروفة" عند التثبيت اليدوي)، أو ابنِ نسخة Release موقّعة لرفعها على Google Play.

### الطباعة والتصدير داخل تطبيق APK

نفس آلية المشاركة الأصلية (Web Share API) المستخدمة في نسخة PWA تعمل أيضاً داخل WebView الخاص بـ Capacitor على أندرويد الحديث، فتفتح قائمة المشاركة/الطباعة الأصلية للنظام دون مغادرة التطبيق. لا حاجة لأي إعداد إضافي.

إن أردت تجربة أعمق (مثل حفظ الملفات مباشرة في مجلد "Downloads" الخاص بالجهاز دون المرور بقائمة المشاركة)، يمكن لاحقاً إضافة حزمتي `@capacitor/filesystem` و `@capacitor/share` الرسميتين — أخبرني إن احتجت ذلك وسأجهز الكود اللازم.
